Stage 6: the LTS timeline. "Which Java version do you use, and what came with it" is a guaranteed interview question, so let's organize everything around the four releases companies actually run: 8, 11, 17, and 21.

Java 8, 2014 — the functional revolution, and still the baseline you must be totally fluent in. Lambdas and functional interfaces: behavior as data, backed by invokedynamic rather than anonymous classes. The Streams API: declarative, lazy collection processing. Optional, as a return-type contract for absence. Default methods on interfaces — and here's the connection people miss: default methods are HOW streams were possible. Collection dot stream is a default method; without them, adding stream to every collection would have broken every implementor on the planet. Also: java dot time, the immutable replacement for the broken Date and Calendar. CompletableFuture for async composition. And on the JVM side, PermGen was removed and Metaspace introduced — class metadata moved to native memory.

Java 11, 2018 — the cleanup LTS, bundling 9 and 10. The module system arrived in 9: most applications stay on the classpath, but the JDK itself got modularized, which set up the later lockdown of internal APIs. Var, from 10 — compile-time inference only, identical bytecode. The standard HTTP client: HTTP/2, async, WebSockets — no more Apache HttpClient dependency. List dot of and Map dot of immutable factories. String got strip, isBlank, lines, repeat. Single-file execution: java Script dot java just runs. The migration pain: Java EE modules — JAXB, JAX-WS — were removed; you re-add them as dependencies. And a great interview detail: JFR became free in 11. In Java 8 it was a commercial Oracle feature; that's why always-on production profiling only became standard practice after 11.

Java 17, 2021 — the data-modeling LTS, bundling 12 through 16. This is today's lesson: records from 16, sealed classes from 17, pattern matching for instanceof from 16, switch expressions from 14, text blocks from 15. Plus helpful NullPointerExceptions — the JVM now tells you exactly which variable in a chain was null. The migration pain: strong encapsulation became enforced by default. Reflective hacks into JDK internals need explicit add-opens flags, which is why old versions of Lombok and Mockito broke on 17. JVM side: ZGC and Shenandoah went production-ready.

Java 21, 2023 — the concurrency LTS. Virtual threads, finalized: millions of cheap threads, blocking code that scales like async, thread-per-request made sane again. Pattern matching for switch and record patterns, finalized. Sequenced collections. Structured concurrency and scoped values in preview — scoped values being the ThreadLocal replacement designed for a world of a million virtual threads. And generational ZGC: sub-millisecond pauses now with generational efficiency, tying right back to Day 4.

Here's your one-line-per-LTS summary for interviews. Eight changed how we write Java — lambdas and streams. Eleven cleaned the platform — modules, HTTP client, free JFR. Seventeen changed how we model data — records, sealed, pattern matching. Twenty-one changed how we do concurrency — virtual threads and structured concurrency. Attach two concrete features and one migration pain to each sentence, and you've answered the version question at a senior level.
