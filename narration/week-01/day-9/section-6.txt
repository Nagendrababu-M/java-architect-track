Stage 6: the same four LTS releases as Day 8, but through today's lens — how the streams and functional toolkit evolved. Interviewers use this to date your experience. A candidate still writing collect Collectors toList on Java 17 is telegraphing that they stopped learning in 2014.

Java 8 invented the model. Lambdas with the invokedynamic machinery. The java util function package — the core four plus about forty primitive specializations. Streams version one: filter, map, reduce, collect, the spliterator engine, parallel streams. Collectors version one: toList, toMap, joining, groupingBy. Optional version one — map, flatMap, filter, orElse. But know the gaps, because the gaps are interview material: no takeWhile, so you couldn't conditionally stop a stream. Stream iterate was infinite-only. And bridging Optional into a stream required an ugly workaround.

Java 11 — bundling 9 and 10 — filled those gaps. TakeWhile and dropWhile: conditional truncation, which is what makes the pagination pattern from Stage 5 possible. The three-argument Stream iterate: seed, condition, step — a for-loop as a stream, finally bounded. Stream ofNullable. Optional grew up: or, for lazy alternative sources; ifPresentOrElse; stream — enabling the flatMap Optional-colon-colon-stream idiom; and isEmpty. Collectors gained flatMapping and filtering as downstream collectors — the keep-empty-groups pattern from Stage 3. And two small gems: Predicate dot not, turning a positive method reference negative, and String dot lines.

Java 17 — bundling 12 through 16 — polished ergonomics. Stream dot toList: shorter, and critically, it returns an UNMODIFIABLE list, where Collectors toList happened to give you a mutable ArrayList. Mechanically migrating code that mutates the result is a real production break — that subtlety is a favorite interview probe. MapMulti, the allocation-free flatMap alternative. Collectors teeing from 12. And an underrated one: records as stream tuples. Multi-value intermediate shapes used to mean SimpleEntry or a hand-written class; local records — declared right inside a method — made one-off pipeline shapes free.

Java 21, plus 22's Gatherers, drew the boundaries. Virtual threads hardened the parallel stream rule: parallel streams for CPU-bound work, virtual threads for I/O-bound fan-out. Before 21, people abused parallel streams for I/O because platform threads were expensive — that excuse is gone. Sequenced collections meet streams: map reversed entrySet stream, descending iteration with no copying. Record patterns bring Day 8's exhaustive switch inside your map operations. And Gatherers close the final gap: custom intermediate operations — windowFixed, windowSliding, scan — things Collectors could never do because they only customize the terminal step.

The evolution in one sentence per release: Eight invented the model. Eleven filled the gaps. Seventeen polished the ergonomics. Twenty-one and twenty-two drew the concurrency boundary and opened the pipeline itself. Next stage: the capstone challenge.
